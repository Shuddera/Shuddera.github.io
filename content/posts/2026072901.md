## 流式对话示例——让它像真人一样打字

上面是非流式响应的基本用法。如果你想实现那种“一个字一个字打出来”的效果（就跟在ChatGPT网页版里看到的一样），用流式输出就行。

javascript

import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: '你的千聚API Key',
});

async function main() {
  const stream = await client.chat.completions.create({
    model: 'grok-3',
    messages: [
      { role: 'user', content: '用幽默的语气，解释一下什么是云计算' }
    ],
    stream: true, // 开启流式输出
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}

main();

运行后，你会看到Grok3像真人一样逐字输出回答，体验非常流畅。这就是千聚国内直连的威力——延迟低到几乎感觉不到。

## 多轮对话示例——构建带记忆的机器人

实际项目中，你往往需要维持对话上下文。下面这个例子展示了如何实现多轮对话：

javascript

import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: '你的千聚API Key',
});

async function chatWithMemory() {
  const messages = [
    { role: 'system', content: '你是一个友好的助手，说话简洁直接。' },
    { role: 'user', content: '我最近在学习Node.js，有什么入门建议？' }
  ];

  // 第一轮回复
  let response = await client.chat.completions.create({
    model: 'grok-3',
    messages: messages,
  });
  const reply1 = response.choices[0].message.content;
  console.log('助手:', reply1);

  // 把历史对话加入上下文
  messages.push({ role: 'assistant', content: reply1 });
  messages.push({ role: 'user', content: '能给我一个简单的异步编程例子吗？' });

  // 第二轮回复，带上下文
  response = await client.chat.completions.create({
    model: 'grok-3',
    messages: messages,
  });
  console.log('助手:', response.choices[0].message.content);
}

chatWithMemory();

Grok3能准确理解上文的对话背景，给出连贯的回答。这对于构建像客服机器人、学习助手这类应用来说，是必不可少的能力。
