在你的工作目录下，打开终端，执行以下命令创建一个Node.js项目：

然后安装 `openai` 官方库，虽然我们调用的不是OpenAI官方，但千聚的接口是完全兼容OpenAI格式的，所以直接用 `openai` 包就好：

`dotenv`用于安全地管理你的API Key，避免硬编码到代码里。

### 第二步：设置环境变量

在项目根目录下创建 `.env` 文件，写入：

请务必把“你的千聚API Key”换成刚才复制的那一串。这个文件不要提交到Git，保证安全。

### 第三步：编写核心调用代码

创建 `index.js` 文件，写入以下代码：

javascript
// 引入依赖
import OpenAI from 'openai';
import 'dotenv/config';

// 关键的配置行：这里只需要改 baseURL
const openai = new OpenAI({
    apiKey: process.env.GROK_API_KEY,
    baseURL: 'https://www.qianjuai.com/v1' // 直接指定千聚的API入口
});

async function callGrok4Fast() {
    try {
        const completion = await openai.chat.completions.create({
            model: "grok-4-fast", // 模型名称，请确认千聚控制台的准确名称
            messages: [
                { role: "system", content: "你是Grok4Fast，一个高效的全能AI助手。" },
                { role: "user", content: "用Node.js写一个读取当前目录下所有文件并列出文件大小的函数" }
            ],
            max_tokens: 1024,
            temperature: 0.7,
        });

        console.log("Grok4Fast的回复：");
        console.log(completion.choices[0].message.content);
    } catch (error) {
        console.error("调用出错:", error.message);
    }
}

callGrok4Fast();

### 进阶：流式输出（Streaming）

对于编程助手这类应用，流式输出体验更好，用户看到结果不用等太久。代码也只需要稍微改动一下：

javascript
async function streamGrok4Fast() {
    const stream = await openai.chat.completions.create({
        model: "grok-4-fast",
        messages: [
            { role: "user", content: "详细解释一下JavaScript中的闭包概念，并用代码举例。" }
        ],
        stream: true, // 开启流式输出
    });

    for await (const chunk of stream) {
        process.stdout.write(chunk.choices[0]?.delta?.content || "");
    }
}

streamGrok4Fast();


这就实现了逐字输出，效果和在官网聊天界面看到的一模一样。千聚平台对流式输出的支持很稳定，且完全没有并发限制。
