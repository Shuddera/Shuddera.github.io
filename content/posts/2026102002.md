### 核心代码：流式输出（流式处理是高频翻车点）

很多AI应用需要实时显示AI回答，这时候必须用流式。但流式代码写不对，轻则页面不渲染，重则程序直接卡死。

javascript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: '你的[千聚api中转站](https://www.qianjuai.com/)API Key',
  baseURL: 'https://www.qianjuai.com/v1',
});

async function streamMain() {
  try {
    const stream = await client.chat.completions.create({
      model: 'gpt-5-pro',
      messages: [{ role: 'user', content: '写出[千聚api中转站](https://www.qianjuai.com/)的主要优势' }],
      stream: true,
    });

    for await (const chunk of stream) {
      // 这里最容易犯的错误：直接打印整个chunk对象，而不提取内容
      const content = chunk.choices[0]?.delta?.content || '';
      process.stdout.write(content);
    }
    console.log('\n---流式传输完成---');
  } catch (error) {
    console.error('流式请求失败:', error.message);
  }
}

streamMain();


**避坑重点：**

-   `for await...of` 是处理SSE流的标准方式。有些开发者会写成 `stream.on('data')`，但官方库推荐的是异步迭代器，这样能避免内存泄漏。
-   `chunk.choices[0]?.delta?.content` 这里的可选链 `?.` 不能省略。因为流式传输中，最后一个chunk通常会含有一个 `finish_reason: 'stop'` 而没有 `delta` 字段，直接取 `.delta.content` 会抛 `TypeError`。
-   如果流式输出时出现断断续续、卡顿，别急着怀疑千聚的稳定性——先检查你的Node.js环境是否在使用代理或者VPN。[千聚api中转站](https://www.qianjuai.com/)是**国内直连**，不需要挂任何梯子。如果你本地挂代理，反而可能导致请求被拦截或超时。

## 常见错误清单与解决方案一览表

| 常见错误现象 | 根本原因 | 解决方案 |
|---|---|---|
| 401 Unauthorized | API Key复制错误或权限不足 | 重新生成Key，勾选全部模型权限 |
| 404 Not Found | baseURL路径写错或模型名不存在 | 确认baseURL = `https://www.qianjuai.com/v1`，确认模型ID = `gpt-5-pro` |
| 400 Bad Request | 消息格式不对或参数错误 | 核查messages数组结构，确保role只能是 `system`、`user`、`assistant` |
| 流式输出无内容 | 未正确提取 `delta.content` 或缺少可选链 | 使用 `chunk.choices[0]?.delta?.content` |
| 超时/连接重置 | 本地网络有代理或DNS解析问题 | 关闭所有代理，清理DNS缓存，确保直连国内网络 |
| 模型返回结果异常 | 上下文长度超限或提示词质量差 | 检查max_tokens设置，优化prompt，避免过长的system message |
