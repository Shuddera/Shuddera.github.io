## 常见报错与其排查

新手使用过程中，可能会遇到几个小问题。我把最常见的三种报错和解决办法列出来，你对照排查就行。

### API Key错误：认证失败

**表现**：提示 `401 Unauthorized` 或 `Invalid API key`

**原因**：复制Key复制错了，或者Key前面有空格。

**解决**：回到[千聚api聚合平台](https://www.qianjuai.com/)控制台重新复制Key，粘贴时注意首尾不要带多余空格。

### 模型名称错误

**表现**：提示 `404: Model not found` 或 `Unknown model`

**原因**：模型名称写错了，或者没有加正确的分隔符。

**解决**：确认 `model=“gemini-2.5-flash”` 一定是小写，名称里没有多余字符。[千聚api聚合平台](https://www.qianjuai.com/)对模型名称要求严格和官网命名一致，别写成 `Gemini-2.5-Flash` 这种大小写杂糅的写法。

### 网络连接错误

**表现**：提示 `Request timeout` 或 `Connection refused`

**原因**：使用了本地的代理设置干扰了请求，或者是DNS解析问题。

**解决**：在代码运行时关掉系统代理，或者在代码中指定 `no_proxy=‘*’`，让请求直接走国内的[千聚api聚合平台](https://www.qianjuai.com/)节点。如果还是不行，检查一下DNS是否用了特殊配置很复杂的本地自建服务。
