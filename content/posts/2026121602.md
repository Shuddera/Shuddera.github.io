## 鉴权与基础请求——最核心的“改一行代码”环节

千聚的API鉴权方式跟OpenAI一毛一样：用`Bearer Token`，把API Key放在请求头里。下面是完整的Java代码示例：

java
import okhttp3.*;
import com.google.gson.Gson;
import com.google.gson.JsonObject;
import java.io.IOException;

public class O4MiniClient {

    private static final String API_URL = "https://www.qianjuai.com/v1/chat/completions";
    private static final String API_KEY = "你的千聚API Key"; // 换成你自己的

    private final OkHttpClient client;
    private final Gson gson;

    public O4MiniClient() {
        this.client = new OkHttpClient.Builder()
                .connectTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .readTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .build();
        this.gson = new Gson();
    }

    public String sendMessage(String userMessage) throws IOException {
        // 构造请求体
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "o4-mini");
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", userMessage);
        requestBody.add("messages", gson.toJsonTree(new JsonObject[]{message}));
        requestBody.addProperty("max_tokens", 1024); // 可调整
        requestBody.addProperty("temperature", 0.7);

        // 构造HTTP请求
        Request request = new Request.Builder()
                .url(API_URL)
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(requestBody.toString(), MediaType.get("application/json; charset=utf-8")))
                .build();

        // 发送请求并获取响应
        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new IOException("请求失败，状态码：" + response.code() + "，响应体：" + response.body().string());
            }
            // 解析响应
            String responseBody = response.body().string();
            JsonObject jsonResponse = gson.fromJson(responseBody, JsonObject.class);
            // 提取AI回复内容
            String assistantReply = jsonResponse
                    .getAsJsonArray("choices")
                    .get(0)
                    .getAsJsonObject()
                    .getAsJsonObject("message")
                    .get("content")
                    .getAsString();
            return assistantReply;
        }
    }

    public static void main(String[] args) {
        O4MiniClient client = new O4MiniClient();
        try {
            String reply = client.sendMessage("用Java写一个快速排序算法");
            System.out.println("o4-mini 回复：\n" + reply);
        } catch (IOException e) {
            System.err.println("调用失败：" + e.getMessage());
        }
    }
}

## 流式输出（Stream）——实时看到AI“打字”

普通请求是阻塞的，等所有Token生成完了才返回，耗时较长。流式输出（SSE）能让你看到AI在逐字生成回答，用户体验更好。千聚也支持OpenAI兼容的流式接口，在请求体里加一个`"stream": true`就行。

java
public void sendStreamMessage(String userMessage) throws IOException {
    JsonObject requestBody = new JsonObject();
    requestBody.addProperty("model", "o4-mini");
    JsonObject message = new JsonObject();
    message.addProperty("role", "user");
    message.addProperty("content", userMessage);
    requestBody.add("messages", gson.toJsonTree(new JsonObject[]{message}));
    requestBody.addProperty("stream", true);  // 关键

    Request request = new Request.Builder()
            .url(API_URL)
            .addHeader("Authorization", "Bearer " + API_KEY)
            .addHeader("Content-Type", "application/json")
            .post(RequestBody.create(requestBody.toString(), MediaType.get("application/json; charset=utf-8")))
            .build();

    client.newCall(request).enqueue(new Callback() {
        @Override
        public void onFailure(Call call, IOException e) {
            System.err.println("流式请求失败：" + e.getMessage());
        }

        @Override
        public void onResponse(Call call, Response response) throws IOException {
            if (!response.isSuccessful()) {
                System.err.println("请求失败：" + response.code());
                return;
            }
            // 逐行读取流数据
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(response.body().byteStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    if (line.startsWith("data: ")) {
                        String data = line.substring(6);
                        if ("[DONE]".equals(data)) {
                            System.out.println("\n——流式输出结束——");
                            break;
                        }
                        JsonObject json = gson.fromJson(data, JsonObject.class);
                        String content = json
                                .getAsJsonArray("choices")
                                .get(0)
                                .getAsJsonObject()
                                .getAsJsonObject("delta")
                                .get("content")
                                .getAsString();
                        System.out.print(content);
                    }
                }
            }
        }
    });
}


调用方式也简单：`new O4MiniClient().sendStreamMessage("讲个冷笑话");`，你会在控制台看到内容像打字一样逐字输出。

## 鉴权细节与常见坑

千聚的鉴权跟OpenAI一模一样，但有几个点容易踩坑：

1. **API Key格式**：千聚的Key是“sk-”开头的字符串，复制时注意别带空格或换行。
2. **Bearer前缀不能少**：有些同学拼接时忘了空格，比如`"Bearer" + key`，应该写成`"Bearer " + key`（空格不能省）。
3. **请求体模型名必须完全一致**：o4-mini就是`"o4-mini"`，别写成`"o4-mini-2025-01-31"`或乱加后缀。模型列表可以在千聚官网查到。
