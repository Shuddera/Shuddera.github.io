## 高级任务：拦截器（Interceptors）实战——防滥用与权限校验

在企业级的Java应用中，我们不能让每个服务模块拿着API Key随便打外网。最好的做法是：**通过内部网关或OkHttp拦截器，自动给所有请求加上认证头以及限流层，同时对响应进行统一日志记录。**

下面是一个标准的使用OkHttp拦截器的实现，完美适配[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)的API调用：

java
// 拦截器核心逻辑
public class ApiAuthInterceptor implements okhttp3.Interceptor {
    private final String globalApiKey;

    public ApiAuthInterceptor(String globalApiKey) {
        this.globalApiKey = globalApiKey; // 从[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)获取的统一Key
    }

    @Override
    public okhttp3.Response intercept(Chain chain) throws IOException {
        okhttp3.Request originalRequest = chain.request();

        // 设置全局认证头
        okhttp3.Request authorizedRequest = originalRequest.newBuilder()
                .header("Authorization", "Bearer " + this.globalApiKey)
                .header("Content-Type", "application/json")
                .header("X-Custom-Tag", "java-team")  // 自定义标记，便于千聚官网查账
                .method(originalRequest.method(), originalRequest.body())
                .build();

        // 日志输出（审计）
        System.out.println("【千聚ai】请求：" + originalRequest.url());

        okhttp3.Response response = chain.proceed(authorizedRequest);
        return response;
    }
}

## 稳如老狗：重试机制（Retry）与优雅降级

调用GPT-5这类高端模型API时，最常见的问题就是突发的高并发导致503或429错误。[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)虽然提供99.9%的可用性，但在上游模型无法避免限流时，我们必须有一套合理且节约Token的重试策略。

下面是一个带指数退避和最大重试限制的重试器：

java
import java.util.concurrent.TimeUnit;

public class TokenRetryHandler {
    
    public static CompletionResult invokeWithRetry(
            int maxRetries, 
            Runnable apiCall) throws InterruptedException {

        int currentRetry = 0;
        CompletionResult result;

        while (currentRetry <= maxRetries) {
            try {
                result = apiCall(); // 执行你的OpenAiService具体方法
                return result; // 成功，直接返回
            } catch (Exception e) {
                if (currentRetry == maxRetries) {
                    System.out.println("已达最大重试次数 " + maxRetries + " 次，返回默认结果！");
                    return CompletionResult.fallback(); // 返回默认降级结果
                }
                
                // 指数退避：每次重试等待 2 的指数次方秒
                long sleepSeconds = (long) Math.pow(2, currentRetry);
                System.out.println("请求失败，等待 " + sleepSeconds + " 秒后重试...原因：" + e.getMessage());
                TimeUnit.SECONDS.sleep(sleepSeconds);
                currentRetry++;
            }
        }
        // 不会走到这里
        return null;
    }
}


**成本控制提醒**：建议将`maxRetries`控制在3以内，因为每次重试都在消耗请求配额（非Token消耗，但会造成额外网络波动）。配合[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)提供的余额实时查询接口，在代码中设置余额低于10元时触发自动熔断，再也不怕一觉醒来欠费上万。
