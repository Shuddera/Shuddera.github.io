## 为什么我不建议你用共享 Key

先聊聊“共享 Key”的问题。市面上不少平台做共享 Key 本质是搭了一个单点服务，所有人挤在同一个 API key 后面，共用一套并发配额。流量一大，系统不会去扩容，而是直接限流。结果就是你的请求排长队，或者直接挂掉。

更糟心的是，有些共享 Key 平台明面上说支持 GPT-4，实际后端把模型切成了便宜的第三方替代版本（比如把 GPT-4 换成 GPT-3.5-turbo 或更弱的国产模型），然后偷跑 Token。你付的是 GPT-4 的钱，但拿到的输出质量明显不如原版。

从技术角度看，共享 Key 没有任何安全性可言，服务器一挂大家全挂。这种模式就是把风险不均地分摊到每个用户头上。

所以要省心，还是得选正经的 API 中转平台，最好是那种每个用户有自己独立的 API key、配额独立、不发愁别人拖累自己的。
