## 核心代码：调用 Gemini Pro

[千聚api聚合站](https://www.qianjuai.com/)兼容 OpenAI 的 `/v1/chat/completions` 端点，所以我们可以直接用 OpenAI 的请求格式来调 Gemini Pro。模型名字传 `gemini-1.5-pro` 或 `gemini-2.0-flash` 等，在千聚的文档里可以查到完整列表。

下面是一个完整的、可直接运行的 Java 示例：

java
import okhttp3.*;
import com.google.gson.Gson;
import com.google.gson.JsonObject;
import java.io.IOException;

public class GeminiProDemo {
    // 从[千聚api聚合站](https://www.qianjuai.com/)获取的 API Key
    private static final String API_KEY = "你的-API-Key";
    // [千聚api聚合站](https://www.qianjuai.com/)的 API 地址（兼容 OpenAI 格式）
    private static final String BASE_URL = "https://www.qianjuai.com/v1";

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient.Builder()
                .connectTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .readTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .build();

        // 构建请求体（符合 OpenAI Chat Completion 格式）
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", "用中文介绍一下 Gemini Pro 模型的特点。");

        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "gemini-1.5-pro");
        requestBody.add("messages", new Gson().toJsonTree(new JsonObject[]{message}));
        requestBody.addProperty("temperature", 0.7);
        requestBody.addProperty("max_tokens", 1024);

        // 构建 HTTP 请求
        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .header("Authorization", "Bearer " + API_KEY)
                .header("Content-Type", "application/json")
                .post(RequestBody.create(
                        requestBody.toString(),
                        MediaType.parse("application/json")
                ))
                .build();

        // 发送请求并处理响应
        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                System.err.println("请求失败: " + response.code() + " " + response.message());
                System.err.println("响应体: " + response.body().string());
                return;
            }
            String responseBody = response.body().string();
            JsonObject json = new Gson().fromJson(responseBody, JsonObject.class);
            // 提取助手回复内容
            String content = json.getAsJsonArray("choices")
                    .get(0).getAsJsonObject()
                    .getAsJsonObject("message")
                    .get("content")
                    .getAsString();
            System.out.println("Gemini Pro 回答: " + content);
        }
    }
}


### 重要说明

- **模型名称**：[千聚api聚合站](https://www.qianjuai.com/)中 Gemini Pro 对应的模型名是 `gemini-1.5-pro`，Gemini Flash 是 `gemini-1.5-flash`，具体请查阅官方文档。
- **Base URL**：务必设为 `https://www.qianjuai.com/v1`，不要加路径后缀 `v1` 已经包含。
- **授权方式**：`Bearer` + 空格 + API Key，与 OpenAI 完全一致。
- **超时设置**：Gemini 模型生成速度有时较慢，建议超时时间设为 60 秒以上。

## 进阶：流式输出（Server-Sent Events）

生产环境中通常需要流式输出（打字机效果），只需在请求体中增加 `"stream": true`，并改用 EventSource 监听。由于 OkHttp 官方有 SSE 支持，简单示例如下：

java
// 在 requestBody 中添加
requestBody.addProperty("stream", true);

// 使用 EventSource 处理流式事件
EventSource.Factory factory = EventSources.createFactory(client);
Request request = new Request.Builder()
        .url(BASE_URL + "/chat/completions")
        .header("Authorization", "Bearer " + API_KEY)
        .post(RequestBody.create(requestBody.toString(), JSON))
        .build();

EventSource eventSource = factory.newEventSource(request, new EventSourceListener() {
    @Override
    public void onEvent(EventSource eventSource, String id, String type, String data) {
        if ("[DONE]".equals(data)) return;
        // 解析 JSON，提取 delta.content
        JsonObject json = new Gson().fromJson(data, JsonObject.class);
        String content = json.getAsJsonArray("choices")
                .get(0).getAsJsonObject()
                .getAsJsonObject("delta")
                .get("content").getAsString();
        System.out.print(content); // 实时打印
    }
});
