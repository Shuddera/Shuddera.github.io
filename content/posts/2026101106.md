## 第三步：完整代码示例

以下是一份可以直接复制粘贴、在Python 3.7+环境下运行的完整代码。请你根据代码内的注释，将`YOUR_API_KEY_HERE`替换为你刚刚复制的API Key。

python
import os
from openai import OpenAI

# 重要：替换为你在千聚后台获取的API Key
api_key = "YOUR_API_KEY_HERE" 

# 初始化客户端，关键一步：修改base_url为千聚的接口地址
client = OpenAI(
    api_key=api_key,
    base_url="https://www.qianjuai.com/v1"
)

def chat_with_glm(prompt, model="glm-4v-plus-0111"):
    """
    使用GLM-4.5模型进行对话
    :param prompt: 用户输入的文本
    :param model: 模型名称，默认使用GLM-4V-Plus-0111，也支持glm-4v-plus等
    :return: 模型的回复
    """
    try:
        response = client.chat.completions.create(
            model=model,  # 模型名称，GLM-4.5系列模型在千聚上可直接调用
            messages=[
                {"role": "system", "content": "你是一个有用的AI助手。"},
                {"role": "user", "content": prompt}
            ],
            max_tokens=1024,  # 最大输出长度
            temperature=0.8,  # 温度，控制生成随机性，0-2之间
            stream=False      # 是否流式输出，这里用普通输出，更直观
        )
        # 提取并返回模型回复内容
        return response.choices[0].message.content
    except Exception as e:
        return f"调用API时发生错误：{e}"

# 以下是主程序入口
if __name__ == "__main__":
    # 示例1：基本问答
    question = "请用Python写一个快速排序算法的示例。"
    print(f"用户问题：{question}")
    answer = chat_with_glm(question)
    print(f"GLM-4.5回答：\n{answer}")
    print("-" * 50)
    
    # 示例2：多轮对话（你可以通过维护messages列表来实现）

            break

### 坑3：模型名称写错
GLM-4.5系列在千聚上有多种变体，如`glm-4v-plus-0111`、`glm-4v-plus`等。如果模型名写错，会返回“模型不存在”的错误。你可以在千聚的后台模型列表页确认最新的模型名称。

### 坑4：防火墙或代理影响

### 坑5：输出乱码或截断
如果返回的文字是乱码，检查一下你的Python文件编码，确保文件头加上`# -*- coding: utf-8 -*-`。如果回答被截断，是`max_tokens`设得太小，增加这个参数值即可（比如2048）。

## 进阶：流式输出与大视觉解析

上面的代码是普通非流式输出，适合快速调试。如果你想要类似ChatGPT那种逐字输出的体验，可以用流式模式：
