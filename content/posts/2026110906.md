## 第 3 步：开始写那“3行代码”

打开你的 Python 编辑器（VS Code、PyCharm 或者随便一个记事本都行），新建一个文件，比如叫 `chat_o4_mini.py`，然后写入以下完整代码：

python
from openai import OpenAI

client = OpenAI(api_key="你的API密钥", base_url="https://www.qianjuai.com/v1")

response = client.chat.completions.create(model="o4-mini", messages=[{"role": "user", "content": "你好，请用中文介绍一下你自己。"}])

print(response.choices[0].message.content)


现在，我们来拆解一下这 3 行核心代码到底做了什么：

1.  **`client = OpenAI(...)`**：这行代码干的事很简单，就是**创建了一个“通信管道”**。它告诉系统两件事：你是谁（通过 `api_key`）以及你要往哪发请求（通过 `base_url`）。把 `base_url` 设为千聚的地址，就完成了国内直连，完全不需要翻墙。
2.  **`response = client.chat.completions.create(...)`**：这是真正的“对话”发起命令。它构建了一个聊天请求，参数包括使用哪个模型 (`model="o4-mini"`)，以及发送给模型的消息内容 (`messages`)。这个过程相当于你向模型喊了一句话：“嘿，我用 o4-mini 模型，回答这个问题。”
3.  **`print(response.choices[0].message.content)`**：模型说完话后，会返回一个结构体（可以理解为一个大文件）。你需要指定：“从返回结果里，取出第一个选择项（`choices[0]`），再取里面的消息正文内容（`.message.content`），然后打印出来。”简单说就是：把模型的回答提取出来，显示在屏幕上。

是不是比你想象中要简单得多？没有复杂的封装、没有十几行的 try-catch，这 3 行完成了从“输入问题”到“看到回答”的完整闭环。

## 处理多轮对话：从“一问一答”到“唠嗑”

上面我们只实现了最基础的“一问一答”。如果要持续对话（像 ChatGPT 那样记住上下文），只需要把 `messages` 这个列表维护起来，把每次的交互记录添加进去。

完整的多轮对话代码示意：

python
from openai import OpenAI

client = OpenAI(api_key="你的API密钥", base_url="https://www.qianjuai.com/v1")

# 初始化对话历史
conversation_history = [
    {"role": "system", "content": "你是一个友好的助手。"},
    {"role": "user", "content": "你好，我的名字是小明。"}
]

# 第一次调用，小明自我介绍
response = client.chat.completions.create(model="o4-mini", messages=conversation_history)
reply = response.choices[0].message.content
print(f"AI：{reply}")  # AI应该会说：“你好小明，很高兴认识你！”

# 把第一次的交互加入历史
conversation_history.append({"role": "assistant", "content": reply})
conversation_history.append({"role": "user", "content": "你还记得我叫什么名字吗？"})

# 第二次调用，测试模型是否记得小明
response = client.chat.completions.create(model="o4-mini", messages=conversation_history)
reply = response.choices[0].message.content
print(f"AI：{reply}")  # 模型应该回答正确：你的名字叫小明！


多轮对话的实现，本质上也只是**维护消息列表**，核心代码仍然是上面那 3 行的变体。有了这个基础，你可以用它来做聊天机器人、自动客服、角色扮演等高级应用。

## 进阶提高：把它们封装成一个函数（附完整代码）

为了方便日常调用，我们可以把上面的逻辑封装成一个函数。这样无论你想跑多少次对话，都是“一行代码”的事。

以下是封装后的**完整可用代码**，直接复制粘贴就能跑（记得填上你的 API Key）：

python
from openai import OpenAI

def ask_o4_mini(api_key: str, user_message: str, system_prompt: str = "你是一个聪明的助手。") -> str:
    """
    通过[千聚ai聚合站](https://www.qianjuai.com/)调用 o4-mini 模型。
    
    参数：
        api_key (str): 千聚平台申请的 API Key
        user_message (str): 用户输入的问题
        system_prompt (str, 可选): 系统提示词，默认“你是一个聪明的助手。”
    
    返回：
        str: 模型的回复文本
    """
    client = OpenAI(
        api_key=api_key,
        base_url="https://www.qianjuai.com/v1"
    )
    
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_message}
    ]
    
    response = client.chat.completions.create(
        model="o4-mini",
        messages=messages
    )
    
    return response.choices[0].message.content

# 使用示例
if __name__ == "__main__":
    YOUR_API_KEY = "sk-你的API密钥"
    result = ask_o4_mini(YOUR_API_KEY, "请用一句话解释量子纠缠。")
    print(result)
