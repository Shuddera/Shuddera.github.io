## 核心参数全解读：从基础到进阶

下面这一部分很关键，我会把[千聚api聚合站](https://www.qianjuai.com/)支持的主要参数逐一拆解，配合Java代码示例说明。

### 1. model（模型名称）

千聚支持500+模型，你可以把模型名称直接传进来。例如调用 `gpt-4o`：

java
CompletionRequest request = CompletionRequest.builder()
    .model("gpt-4o")
    .prompt("用Java写一个快速排序")
    .maxTokens(1024)
    .build();


### 2. messages（消息列表）

对话式模型用 `ChatCompletionRequest`：

java
List<ChatMessage> messages = new ArrayList<>();
messages.add(new ChatMessage("system", "你是一个Java代码助手。"));
messages.add(new ChatMessage("user", "帮我写一个单例模式。"));

ChatCompletionRequest chatRequest = ChatCompletionRequest.builder()
    .model("claude-3-5-sonnet-20241022")
    .messages(messages)
    .temperature(0.7)
    .build();


### 3. max_tokens（最大Token数）

控制单次响应长度，建议合理设置避免超时或浪费额度：

java
.maxTokens(2048)


### 4. temperature（温度）

控制响应随机性，0到2之间的浮点数。代码生成场景建议设为0.2，创意写作用0.8。

java
.temperature(0.2)


### 5. top_p（核采样）

与temperature协作，典型值为0.9。如果temperature设为1，则top_p完全生效。

java
.topP(0.9)


### 6. frequency_penalty / presence_penalty

防止模型喋喋不休重复内容：

java
.frequencyPenalty(0.5)
.presencePenalty(0.2)


### 7. stream（流式输出）

流式输出可以实时看到Token冒出来，Java里通过 `Flowable` 处理：

java
service.streamChatCompletion(chatRequest)
    .doOnNext(chunk -> System.out.print(chunk.getChoices().get(0).getMessage().getContent()))
    .blockingSubscribe();

## 完整可运行的Java示例代码

我整理了一个可直接运行的示例类，集成了所有上述参数：

java
import com.theokanning.openai.completion.chat.*;
import com.theokanning.openai.service.OpenAiService;
import java.time.Duration;
import java.util.*;

public class QianjuApiDemo {
    public static void main(String[] args) {
        OpenAiService service = new OpenAiService(
            "sk-你的密钥",
            Duration.ofSeconds(30)
        );
        // 设置[千聚api聚合站](https://www.qianjuai.com/)地址
        service.setBaseUrl("https://www.qianjuai.com/v1");

        List<ChatMessage> messages = Arrays.asList(
            new ChatMessage("system", "你是一名资深Java架构师。"),
            new ChatMessage("user", "帮我解释Java内存模型")
        );

        ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model("gpt-4o")
            .messages(messages)
            .maxTokens(1024)
            .temperature(0.3)
            .topP(0.9)
            .frequencyPenalty(0.2)
            .presencePenalty(0.1)
            .build();

        // 非流式调用
        ChatCompletionResult result = service.createChatCompletion(request);
        String reply = result.getChoices().get(0).getMessage().getContent();
        System.out.println("模型回复：");
        System.out.println(reply);
    }
}

## 异常处理最佳实践

很多开发者在接入过程中最容易卡在异常处理上。下面是[千聚api聚合站](https://www.qianjuai.com/)对接Java场景中最常见的异常类型及处理方法：

### 1. HTTP 401 Unauthorized

**原因**：API Key不正确或已过期。

**处理方案**：
java
try {
    result = service.createChatCompletion(chatRequest);
} catch (OpenAiHttpException e) {
    if (e.statusCode == 401) {
        System.err.println("API Key无效，请前往[千聚api聚合站](https://www.qianjuai.com/)重新生成：https://www.qianjuai.com/register");
    }
}


### 2. HTTP 429 Rate Limit

**原因**：短时间内请求频率太高。

**处理方案**：增加指数退避重试逻辑。

java
int retries = 0;
while (retries < 3) {
    try {
        result = service.createChatCompletion(chatRequest);
        break;
    } catch (OpenAiHttpException e) {
        if (e.statusCode == 429) {
            long wait = (long) Math.pow(2, retries) * 1000;
            Thread.sleep(wait);
            retries++;
        } else {
            throw e;
        }
    }
}


### 3. HTTP 503 Service Unavailable

**原因**：模型过载或网络抖动。

**处理方案**：加上重试 + 日志记录，建议最多重试3次。

### 4. SocketTimeoutException

**原因**：网络连接超时，尤其是首次建立连接时。

**处理方案**：增大 `Duration.ofSeconds()` 参数，建议设为60秒：

java
OpenAiService service = new OpenAiService("sk-密钥", Duration.ofSeconds(60));


### 5. JSON解析异常

**原因**：请求报文格式错误，比如messages结构不对。

**处理方案**：先打印出完整请求体，对照[千聚api聚合站](https://www.qianjuai.com/)官方文档核实字段。

java
ObjectMapper mapper = new ObjectMapper();
System.out.println(mapper.writeValueAsString(chatRequest));
