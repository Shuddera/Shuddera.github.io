## 隐形消费到底是什么，在哪里藏着

先说清楚两个字：**转义**。



很多低代码接入教程，会让用户直接对着官方 API 调用 Qwen3-Max。这本身没毛病，但问题出在“调用链”上。

比如，在一个标准的低代码平台里，你要接入一个模型，通常要经过这个流程：

1.  低代码平台 → 2. 它的 API 网关 → 3. 大模型厂商的 API → 4. 返回结果。

这里面，第 2 步的“API 网关”就是最大的隐形消费来源。很多低代码平台自己并不生产模型，它们只是作为一个中间层转发你的请求。但它们在转发时，会进行所谓的“转义处理”——把你的输入参数和自己预置的格式打包在一起再发送。

**这个转义本身通常免费，但隐藏着巨大的消耗陷阱。** 比如，你请求一个 500 Token 的任务，经过转义后，实际上传的参数变成了 800 Token，甚至更多。而你被扣费，是按 800 Token 计算的。换句话说，你为 300 个压根没用上的 Token 付了钱。

这就是“转义通胀”。在你每天调用几百次 API 的情况下，一个月下来，凭空多花 50% 甚至 80% 的钱都是可能的。不少人以为是模型本身涨价了，其实是被中间层偷偷多收了。
