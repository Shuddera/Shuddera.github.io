## 为什么会遇到429

先拆一下429的本质。大多数API服务都有限流机制，比如你调用千聚AI官网的接口，免费或低等级账号可能只允许每秒1次请求。这个限制不是针对你，是为了保护后端不被冲垮。

但问题在于，**现在的很多应用场景——比如批量处理文本、实时数据分析、AI内容生成——天生就需要高并发**。如果你只有一个请求通道，每秒1次，等你处理完1000条数据，25分钟已经过去了。而如果你能提到100次/秒，同样的任务10秒搞定。

**429的根因，从来不是“你的程序有问题”，而是“架构层面缺乏对限流的应对策略”。**

def call_model_with_retry(prompt, retries=3):

                "messages": [{"role": "user", "content": prompt}]
            }
            resp = requests.post(url, json=payload, headers=headers, timeout=10)
            if resp.status_code == 429:
                sleep_time = random.uniform(0.1, 0.5)
                time.sleep(sleep_time)

        except Exception as e:
            print(f"Error: {e}")

    return None
